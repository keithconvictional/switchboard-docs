How to Build Word Count Switch for Convictional Products

In this guide, we walk through building a Switch that counts the number of words in a product description. The products are pulled from the Convictional API, number of words is calculated, then the results are push under product.Attributes["word_count"] back onto the API.

This code will run on an AWS Lambda. It will be scheduled for once an hour. We could make it webhook based but we will keep it simple for now. If you have a large number of products, you will need to switch to using webhooks or multiple functions. The general flow of data:

Convictional Products API --> AWS Lambda running Switch Code --> Convictional Products API

We are assuming you have an AWS account, the AWS CLI installed, and Switchboard CLI installed.

Codeâ€‹

Let's get started with writing the code. Create a new repository and clone it to your local machine.

git clone git@github.com:your-username/your-repo-name.git
cd your-repo-name

Run the new command:

switchboard new switch

This should take a few seconds, but you should be able to now see files. Open the project in your favourite editor.

First, the Extract step of ETL will pull from the Convictional products API. If you look at the extract/convictional.go, it has GetProductsFromAPI. It comes out of the box. It requires two environment variables: CONVICTIONAL_API_KEY and IS_BUYER. We need to indicate that we want to pull from Convictional compared another source (Ex. SQS). Look at extract/extract.go, and in Multiple, we call GetProductsFromAPI. The switch statement depends on EXTRACT_METHOD environment variable.

The too long, didn't read (TLDR) is that we need to set:

  • Set EXTRACT_METHOD with convictional_api
  • Set CONVICTIONAL_API_KEY with your API key (How to find it)
  • Set IS_BUYER this is true or false depending on your Convictional account.

That's it for pulling product information. Next is the transform step. This goes into transform/transform.go. Here is the updated code:

package transform

import (
    "regexp"
    "strconv"
    "strings"
    "switchboard-module-boilerplate/models"
)

const regex = `<.*?>`
const wordCountAttributeKey = "word_count"

// Transform returns processed flagged, the updated model, and error.
// Products that have already been processed will be returned as false.
func Transform(product models.Product) (bool, models.Product, error) {
    if _, ok := product.Attributes[wordCountAttributeKey]; ok {
        // Already exists
        return false, models.Product{}, nil
    }

    description := stripHtmlRegex(product.Description)

    // Count words
    if product.Attributes == nil {
        product.Attributes = map[string]interface{}{}
    }
    product.Attributes[wordCountAttributeKey] = strconv.Itoa(CountWords(description))

    return true, product, nil
}

// This method uses a regular expresion to remove HTML tags.
func stripHtmlRegex(s string) string {
    r := regexp.MustCompile(regex)
    return r.ReplaceAllString(s, "")
}


func CountWords(s string)  int {
    return len(strings.Fields(s))
}

This code took less than 10 minutes to write. This is why we are using the Switchboard framework.

The third step of our ETL work is the Load. We are loading back into the Convictional API. Look at the load/load.go and specifically Multiple. It loops through each of the products, if its updated pushes a API call to Convictional. We need to set the environment variable for indicating how we want to load, so LOAD_METHOD should be set with convictional_api. In terms of rate limiting, that comes out of the box.

That's it! Your code should be ready to be deployed. I recommend committing this code to version control.

Deployâ€‹

As previously mentioned, we are going to deploy to AWS Lambda. We are going to use the CLI to deploy this code. There are two ways the deploy command captures information; input or file. For input, it will prompt us with questions and you need to answer. Ex. What are the triggers? For file, you make reference to a config yaml file path in the deploy command.

We are going to go through the prompts. The config file is excellent if it is being deployed through a continuous delivery pipeline.

Just a reminder, the CLI command is structured as follows:

switchboard deploy <platform> <flow> <optional config path>

We are going to run:

switchboard deploy aws Convictional-->Func1-->Convictional

This will kick off the deployment of one serverless function.

# What is the name of your Func1? (Alpha-numeric characters only)
> word-count-switch

# What are the triggers for Func1? (Bar delimited, Options: schedule, api, sqs)
> schedule

# What time should Func1 trigger? (Express as Cron: http://www.cronmaker.com/)
> 0 0 0/1 1/1 * ? *

# What environment variables should be set? (When complete, enter "done", Ex. key1:value1)
> LOAD_METHOD:convictional_api
> EXTRACT_METHOD:convictional_api
> CONVICTIONAL_API_KEY:false
> IS_BUYER:false
> CONVICTIONAL_API_KEY:redacted_from_tutorial
> done

Deploying to us-east-2 (default region)...
Done, check it out: https://us-east-2.console.aws.amazon.com/lambda/home?region=us-east-2#/functions/word-count-switch

We are setup on the cloud.
